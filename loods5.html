Breakdownschets

De eerste week moest ik een website kiezen die ik wou namaken en hier een een breakdownschets van maken. Op die manier kreeg ik een beter beeld over in welke vlakken een pagina was opgedeeld en hoe die vlakken waren opgebouwd.

Coderen

Nadat ik mijn breakdownschets had gemaakt wist ik goed hoe ik mijn HTML moest opbouwen. Door notities te gebruiken in mijn HTML en die terug te laten komen in mijn CSS wist ik goed welke onderdelen bij elkaar horen.

De website moest mobile-first gecodeerd worden. Hiervoor heb ik de CSS eerst geschreven voor een mobile device. Om de tablet en desktop versie te coderen heb ik media-query's gebruikt. Ook heb ik gebruik gemaakt van display flex en display grid om onderdelen goed naast of onder elkaar te krijgen.

In week 2 en week 4 heb ik ook een toegankelijkheidstest gedaan. De eerste van de originele site en de 2e van mijn eigen site. Door eerst de test te doen bij de originele website kreeg ik een goed overzicht in hoe ik de site beter kon maken. Ik heb er onder andere voor gekozen om de site een light/dark mode te geven.

In de 4e week heb ik de laatste aanpassingen gebracht aan de HTML, CSS en Javascript en heb ik een werkende, responsive en toegankelijke website opgeleverd.
